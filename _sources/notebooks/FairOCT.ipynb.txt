    "## Loan Approval Dataset\n",
    "\n",
    "In this example, we generate a synthetic dataset related to loan approval decisions. The dataset has 5 features: Income, Credit_Score, Employment_Status, Education_Level, and Previous_Default. The target variable is Loan_Approval, which indicates whether a loan application is approved (1) or denied (0). We also include a protected attribute, Gender, to simulate a fairness-related scenario.\n",
    "\n",
    "When using the FairSPOCT class to learn fair optimal decision trees, there are several key parameters to consider:\n",
    "\n",
    "1. depth: This parameter controls the maximum depth of the decision tree. A larger depth allows for more complex trees, but may lead to overfitting. It's recommended to start with a small depth (e.g., 2 or 3) and gradually increase it while monitoring the performance on a validation set.\n",
    "\n",
    "2. _lambda: This is the regularization parameter that balances the trade-off between accuracy and tree complexity. A higher value of _lambda encourages simpler trees. It's typically set to a small value (e.g., 0.01 or 0.1) to prevent overfitting. You can tune this parameter using cross-validation.\n",
    "\n",
    "3. fairness_bound: This parameter controls the strictness of the fairness constraint. A value of 1 means no fairness constraint is enforced, while smaller values enforce stricter fairness constraints. The choice of fairness_bound depends on the desired level of fairness and the trade-off with accuracy. It's recommended to start with a value close to 1 and gradually decrease it while monitoring the fairness metrics and accuracy.\n",
    "\n",
    "The other parameters in the FairSPOCT class include:\n",
    "\n",
    "- solver: The solver to use for the optimization problem. We use \"gurobi\" in this example, but you can also use \"cbc\".\n",
    "- positive_class: The value of the class label corresponding to the desired outcome. In this case, we set it to 1, representing loan approval.\n",
    "- time_limit: The maximum time (in seconds) allowed for solving the optimization problem.\n",
    "- num_threads: The number of threads the solver should use. If set to None, it will use all available threads.\n",
    "- obj_mode: The objective to be used for learning the optimal decision tree. We set it to \"acc\" to optimize for accuracy, but you can also use \"balance\" for balanced accuracy or even \"weighted\" to specify your own weights for each observation.\n",
    "- verbose: If set to True, the solver will display verbose output during the optimization process.\n"

    "When choosing features for legitimate factors versus predictive features, consider the following:\n",
    "\n",
    "- Legitimate factors should be variables that are deemed acceptable to influence the outcome, even if they may result in differences across protected groups. These factors should be based on domain knowledge and societal norms. For example, in a loan approval scenario, the number of previous loan defaults might be considered a legitimate factor.\n",
    "\n",
    "- Predictive features, on the other hand, are variables that are used to make predictions but should not lead to unfair treatment of protected groups. These features should be carefully selected to avoid perpetuating biases or discrimination. For instance, while 'Age' and 'Income' might be predictive of loan approval, they should not be used in a way that unfairly disadvantages certain protected groups.\n",

    "## Additional Supported Fairness Metrics\n",
    "Now that we have seen how the FairSPOCT class can be used to learn fair optimal classification trees and the impact of the fairness bound on the learned trees and fairness metrics, let's explore the other fairness metrics available in the ODTlearn package.\n"
   ]
  },
  {
   "cell_type": "markdown",
   "metadata": {},
   "source": [
    "In the previous sections, we focused on using the FairSPOCT class to learn fair optimal classification trees with statistical parity constraints. However, the ODTlearn package provides implementations for several other fairness metrics, each capturing different aspects of fairness. These include:\n",
    "\n",
    "- FairCSPOCT: Enforces conditional statistical parity\n",
    "- FairPEOCT: Enforces predictive equality\n",
    "- FairEOppOCT: Enforces equal opportunity\n",
    "- FairEOddsOCT: Enforces equalized odds\n",
    "\n",
    "Each of these classes follows a similar interface to FairSPOCT, with the main difference being the fairness metric they enforce.\n",
    "\n",
    "Here's an overview of each fairness metric and how it is calculated:\n",
    "\n",
    "1. Statistical Parity (FairSPOCT):\n",
    "   - Definition: A classifier satisfies statistical parity if the probability of receiving a positive outcome is equal across all protected groups.\n",
    "   - Equation: $P(\\hat{Y} = 1 | A = a) = P(\\hat{Y} = 1 | A = b), \\forall a, b$\n",
    "   - $\\hat{Y}$ is the predicted outcome, and $A$ is the protected attribute.\n",
    "\n",
    "2. Conditional Statistical Parity (FairCSPOCT):\n",
    "   - Definition: A classifier satisfies conditional statistical parity if the probability of receiving a positive outcome is equal across all protected groups, conditioned on a set of legitimate factors.\n",
    "   - Equation: $P(\\hat{Y} = 1 | A = a, L = l) = P(\\hat{Y} = 1 | A = b, L = l), \\forall a, b, l$\n",
    "   - $\\hat{Y}$ is the predicted outcome, $A$ is the protected attribute, and $L$ represents the legitimate factors.\n",
    "\n",
    "3. Predictive Equality (FairPEOCT):\n",
    "   - Definition: A classifier satisfies predictive equality if the false positive rates are equal across all protected groups.\n",
    "   - Equation: $P(\\hat{Y} = 1 | Y = 0, A = a) = P(\\hat{Y} = 1 | Y = 0, A = b), \\forall a, b$\n",
    "   - $\\hat{Y}$ is the predicted outcome, $Y$ is the true outcome, and $A$ is the protected attribute.\n",
    "\n",
    "4. Equal Opportunity (FairEOppOCT):\n",
    "   - Definition: A classifier satisfies equal opportunity if the true positive rates are equal across all protected groups.\n",
    "   - Equation: $P(\\hat{Y} = 1 | Y = 1, A = a) = P(\\hat{Y} = 1 | Y = 1, A = b), \\forall a, b$\n",
    "   - $\\hat{Y}$ is the predicted outcome, $Y$ is the true outcome, and $A$ is the protected attribute.\n",
    "\n",
    "5. Equalized Odds (FairEOddsOCT):\n",
    "   - Definition: A classifier satisfies equalized odds if both the true positive rates and false positive rates are equal across all protected groups.\n",
    "   - Equation: $P(\\hat{Y} = 1 | Y = y, A = a) = P(\\hat{Y} = 1 | Y = y, A = b), \\forall a, b, y \\in \\{0, 1\\}$\n",
    "   - $\\hat{Y}$ is the predicted outcome, $Y$ is the true outcome, and $A$ is the protected attribute.\n",
    "\n",
    "When applying fairness constraints to a decision tree, it is crucial to carefully choose a fairness metric that aligns with the specific use case and the societal or legal requirements of the problem at hand. Different fairness metrics capture different aspects of fairness and may lead to different trade-offs between fairness and accuracy.\n",
    "\n",
    "For example, if the goal is to ensure that the overall proportion of positive outcomes is similar across protected groups, statistical parity (FairSPOCT) would be an appropriate choice. However, if the focus is on ensuring that the classifier makes similar mistakes across protected groups, predictive equality (FairPEOCT) might be more suitable.\n",
    "\n",
    "It is important to note that achieving fairness in machine learning is a complex and ongoing process that requires careful consideration of the societal context, potential biases in the data, and the limitations of the chosen fairness metric. Engaging with domain experts, stakeholders, and affected communities is essential to understand the specific fairness requirements of the problem and select the most appropriate fairness metric or combination of metrics.\n"
